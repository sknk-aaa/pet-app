# AGENTS.md

このリポジトリは、iPhone 向けペット写真日記アプリ(プロジェクト名未定、以下「本アプリ」)の実装プロジェクトです。Claude Code はこのファイルを最初に読み、続いて `docs/` 配下を順に参照してから実装に着手してください。

---

## プロジェクトの目的

ペットの写真を 1 日 1 枚記録し、カレンダーで見返し、希望者だけが「今日のペット」としてアプリ内に掲載される体験を提供します。

このアプリは健康管理アプリでも SNS でもありません。「1 日 1 枚、今日の渾身の一枚を残す」「カレンダーで成長や思い出を見返す」「連続記録で毎日の習慣にする」「希望した写真だけ、今日のペットとしてアプリ内掲載候補にできる」の 4 点に集中します。

---

## このプロジェクトの構成

実装は次の 3 コンポーネントに分かれます。

1. モバイルアプリ — React Native / Expo (iPhone のみ)
2. 管理画面 — Next.js (Web)
3. バックエンド — Supabase (Auth + Postgres + Storage + Edge Functions + Cron)

モノレポ構成を推奨します。例:

```
/
├── apps/
│   ├── mobile/        モバイルアプリ
│   └── admin/         管理画面
├── supabase/          DB マイグレーション、Edge Functions、cron 定義
├── docs/              本設計書(編集禁止、参照のみ)
└── AGENTS.md          このファイル(編集禁止、参照のみ)
```

ディレクトリ構成の詳細は `docs/07-tech-stack.md` を参照してください。

## 実装方針

- 仕様書に書かれた挙動を正として、勝手な機能追加や仕様変更を行わないでください。
- 仕様書に明記されていない判断が必要な場合は、`docs/01-overview.md` の「設計原則」セクションに従い、迷う場合は実装を止めて確認してください。
- v1.0 のスコープを超える機能 (`docs/08-non-goals.md` 参照) は実装しないでください。
- 技術選定は `docs/07-tech-stack.md` に従ってください。代替案の検討は不要です。
- 仕様書には擬似コードを意図的に含めていません。実装の判断は Claude Code に委ねます。仕様書に書いてあるのは「何を実現すべきか」と「どう振る舞うべきか」です。
