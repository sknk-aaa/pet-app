# 01. 全体像と設計原則

## アプリの位置づけ

本アプリは「ペット写真日記アプリ」として App Store に出します。「ペット管理アプリ」ではありません。多機能なペット管理アプリ(健康管理・支出管理・通院記録・家族共有)とは戦いません。

訴求軸:

- 1 日 1 枚、うちの子を残す
- カレンダーで成長を見返せる
- 毎日の記録が連続日数になる
- 希望した写真だけ、今日のペットに参加できる
- 選ばれたら、みんなからリアクションが届く
- 健康管理ではなく、思い出を残すアプリ

---

## 2 つの核

**第一の核: 1 日 1 枚のペット写真日記**

- 1 日 1 枚だけ写真を記録できる
- タイトルとメモを付けられる
- カレンダーで見返せる
- 連続記録日数が伸びる

**第二の核: 「今日のペット」**

- 希望した写真だけ、アプリ内に「今日のペット」として掲載される可能性がある
- 1 日 1 枚だけ掲載される
- 掲載写真には軽いリアクション(かわいい / きれい / かっこいい / いいね)を押せる

第二の核は SNS ではありません。フォロー、コメント、DM、ランキング、ユーザー同士の交流は入れません。

---

## 用語定義

| 用語 | 定義 |
|---|---|
| 記録 (entry) | ユーザーが 1 日 1 回保存する写真と付随情報(タイトル、メモ、記念日タグ、写っているペット)。端末ローカルに保存 |
| ペット (pet) | ユーザーが登録した自分のペットの情報。無料は 1 ペット、Pro は無制限 |
| 連続記録日数 (display_streak) | ユーザー画面に表示する連続日数。記録を 1 日でも忘れたら 1 にリセット |
| 抽選重み (featured_weight_streak) | 「今日のペット」抽選用の内部連続数。掲載されたら 0 にリセット。display_streak とは別管理 |
| 今日のペット候補 (featured candidate) | ユーザーが「今日のペットに参加」を ON にして投稿した写真。レビュー対象 |
| 今日のペット (featured pet) | 候補から選ばれて、ある日 1 日だけアプリ内に掲載される写真 |
| 記念日タグ (anniversary tag) | 記録に付けられる特別な日のラベル(誕生日 / うちの子記念日 / はじめて / おでかけ / その他) |
| 管理者 (admin) | 管理画面にログインできるユーザー。`users.is_admin = true` で識別 |
| Pro ユーザー | 買い切り Pro または月額 Pro を購入したユーザー |
| device_id | アプリ初回起動時に生成され端末ローカルに保存される UUID。未ログインユーザーの識別に使用 |

## 設計原則

仕様書に明記されていない判断が必要な場合は、以下の原則に従ってください。複数の原則が衝突する場合は、上にあるものを優先してください。

1. **シンプルさを優先する**
   仕様メモに書かれた範囲を超える機能追加は行わない。「便利そう」「あったら良さそう」は v1.0 では入れない。

2. **ユーザーの離脱を防ぐ**
   初回利用のハードルを最小化する。ログインを要求するのは「今日のペットに参加」「通報」「掲載履歴閲覧」「Pro 復元」だけ。それ以外はログイン不要で完結させる。

3. **写真は端末内が原則**
   通常の写真記録は端末ローカルに保存し、サーバーに送らない。サーバーに送るのは「今日のペットに参加」をオンにした写真だけ。

4. **SNS にしない**
   フォロー、コメント、DM、ランキング、いいね通知、人気ペットランキング、ユーザーページ閲覧は入れない。リアクションは「軽い反応」として扱い、承認欲求を煽らない。

5. **UGC は手動レビューを通す**
   今日のペット候補は即公開しない。管理者が手動で承認した写真だけが掲載対象になる。

6. **App Store ガイドラインに従う**
   ユーザー投稿コンテンツを扱うアプリとして、通報機能・非表示機能・ユーザー BAN・利用規約・プライバシーポリシーは必須。

7. **オフラインファースト**
   ログイン不要の機能はすべてオフラインで動作する。サーバー通信が必要な機能(今日のペット参加、リアクション送信、通報送信)は失敗時に送信キューに積み、ネット復帰時に自動再送する。

8. **無料と Pro で UI を分けない**
   Pro 機能のための UI(ペット選択、ペット別フィルター、月まとめなど)は無料ユーザーにも同じ位置に表示する。タップしたら課金画面に誘導する。これにより Pro 機能の存在が自然にユーザーに伝わる。

9. **完成度よりリリースを優先する**
   v1.0 は最小限のスコープで App Store に出すことが目標。完璧を求めない。

10. **メモは絶対にプライベート**
    記録のメモは端末ローカルにしか保存しない。今日のペットへ参加してもサーバーに送らない。掲載画面でも表示しない。

## v1.0 のスコープ

詳細は `docs/08-non-goals.md` を参照してください。

**無料機能:**

- 1 ペット登録
- 1 日 1 枚の写真記録(タイトル、メモ、記念日タグ)
- カレンダー表示
- 連続記録日数
- 通知リマインダー(ローカル通知)
- 記念日一覧
- 思い出表示(去年の今日 → ランダム)
- ログイン(Apple / Google / メール)
- 今日のペット参加・閲覧・リアクション・通報
- 掲載履歴
- ペット選択 UI(タップで Pro 誘導)

**Pro 機能(買い切り ¥1,500 / 月額 ¥400):**

両プラン同一機能。課金は RevenueCat(react-native-purchases)経由で、価格は App Store Connect / RevenueCat の設定値を実行時に取得して表示する。

v1.0 実装済み:

- 複数ペット登録(無制限。無料は 1 ペット)
- 写真への複数ペット紐づけ
- ペット別カレンダー
- ペット別記念日一覧

将来追加候補(v1.0 未実装):

- 月まとめ画像保存
- Before / After 比較
- ウィジェット拡張
- ペット別思い出表示

**入れないもの:**

- 1 日複数枚記録
- 健康管理・支出管理・ごはん記録・トイレ記録・散歩記録・通院管理
- 家族共有
- 詳細グラフ
- SNS 的なフォロー / コメント / DM / ランキング
- リアクション通知
- クラウドバックアップ・同期
- Android 対応

## 想定ユーザーと利用シーン

主なユーザー像:

- ペット(犬・猫を中心に)を飼っている個人
- iPhone で日々ペットを撮影している
- SNS にアップするほどではないが、自分や家族のためにペットの写真を残したい
- 健康管理は別アプリで十分、写真を中心にした思い出記録アプリを求めている

主な利用シーン:

- 朝・夕・寝る前に、その日のペットの写真を 1 枚記録する
- 数日後・数ヶ月後に、カレンダーから過去の写真を見返す
- ときどき「今日のペット」画面を開いて、他のペット写真にリアクションする
- 自分のペットを今日のペットに投稿して、選ばれたら喜ぶ

## デザイン方向性

| 項目 | 方針 |
|---|---|
| 背景 | 白基調、ところどころ薄ベージュ |
| アクセント色 | くすみオレンジ |
| 角丸 | 柔らかい角丸(8〜16px) |
| UI スタイル | iOS 標準コンポーネントに近い、過度な装飾は避ける |
| 情報密度 | 少なめ、余白を活かす |
| 主役 | ペット写真。UI は写真を引き立てる脇役 |

**避けるもの:**

- 肉球モチーフ装飾
- キラキラ・グラデーション過多
- AI 臭い装飾(虹色グラデーション、不必要なエモジ、過剰なイラスト)
- SNS 感(タイムライン的なフィード、いいねカウンター強調)
- インスタライク・LINE ライクな配色
