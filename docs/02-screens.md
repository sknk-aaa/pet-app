# 02. 画面仕様

このドキュメントはユーザーが操作するすべての画面の一覧、画面遷移、各画面の UI 要素と振る舞いを定義します。

---

## タブ構成

3 タブ。タブバーは画面下部に固定。

| 位置 | タブ名 | 内容 |
|---|---|---|
| 左 | カレンダー | 月単位の写真カレンダー |
| 中央 | ホーム | 今日の記録・連続日数・思い出 |
| 右 | 今日のペット | アプリ内に掲載される 1 枚 |

設定画面はホーム画面の右上歯車アイコンから遷移。

## 共通 UI 要素

### ペットフィルター(画面右上)

ホーム・カレンダーの右上にペットフィルターを表示。

**表示内容:**
- 「選択中のペットアイコン」(円形のペット顔写真または既定アイコン)

**動作:**
- タップで「ペット選択シート」を開く
- 無料ユーザーで未登録ペットがある場合も同じ UI を表示

### ペット選択シート

下からモーダル表示。

**表示内容(全ユーザー共通):**

- 「すべて」項目(選択時はすべてのペットの写真を表示)
- 登録済みペット一覧(各行にアイコン・名前)
- 「+ ペットを追加」ボタン

**動作:**
- ペット行タップ → 選択中ペットを切り替え、シートを閉じる
- 「+ ペットを追加」タップ:
  - Pro ユーザー: ペット追加フォーム へ遷移
  - 無料ユーザーで既に 1 ペット登録済: Pro 購入画面へ遷移し、画面トップに「Pro でもっとペットを追加できます」のメッセージ表示
  - 無料ユーザーで未登録: ペット追加フォーム へ遷移

### 設定アイコン(歯車)

ホーム画面の右上にのみ表示。タップで設定画面へ。

## 1. オンボーディング

**用途:** 初回起動時のみ表示。アプリの世界観を 3 〜 4 枚で伝える。

**表示:**

- 横スワイプ式のスライド 3 〜 4 枚
- 各スライド: イラストまたは写真風グラフィック + 短いキャッチコピー
- 例:
  - 「1 日 1 枚、うちの子を残す」
  - 「カレンダーで成長を見返せる」
  - 「希望した写真だけ、今日のペットに参加できる」
- 最後のスライドに「はじめる」ボタン

**動作:**
- 「はじめる」タップ → ペット初期設定画面へ
- スライドは「スキップ」可能(右上にリンク)。スキップでもペット初期設定画面へ
- 一度完了したら二度と表示しない(ローカルフラグで記憶)

---

## 2. ペット初期設定

**用途:** 初回起動時に必ず最初のペットを登録させる。

**入力項目:**

| 項目 | 必須/任意 | 形式 |
|---|---|---|
| 名前 | 必須 | テキスト(最大 20 文字) |
| 種類 | 必須 | 選択: ねこ / いぬ / インコ / うさぎ / ハムスター / その他 |
| 種類(その他の場合の自由入力) | 上が「その他」の時のみ必須 | テキスト(最大 20 文字) |
| 誕生日または年齢 | 任意 | 日付 or 年齢(数字) |
| 性別 | 任意 | 選択: オス / メス / 不明 |
| アイコン画像 | 任意 | カメラ撮影 or アルバム選択。未設定時は種類別の既定アイコン |

**動作:**
- 「完了」タップ → ホーム画面(タブ) へ
- このとき登録したペットを「選択中ペット」として保存
- 通知許可ダイアログをこのタイミングで表示(iOS 標準ダイアログ)

## 3. カレンダー(タブ)

**用途:** 月単位で記録を見返す。

**画面構成:**

```
┌────────────────────────────────────┐
│ < 2026年5月 >          [🐾アイコン] │ ← 月切替 + ペットフィルター
├────────────────────────────────────┤
│ [🎀 記念日を見る]                  │ ← 思い出ゾーン
├────────────────────────────────────┤
│  日 月 火 水 木 金 土               │ ← 曜日ヘッダー
│                                    │
│  各セルに:                          │
│   - 日付                            │
│   - 写真サムネイル (あれば)         │
│   - 🎀 記念日マーク (右上)          │
│   - ⭐ 掲載マーク (左上)            │
└────────────────────────────────────┘
```

**動作:**

- 月切替: 左右の矢印 or 左右スワイプ
- 日付セルタップ:
  - 写真があれば 日別詳細 へ
  - 写真がなく当日: 写真記録フォーム へ
  - 写真がなく当日以外: 何もしない(または「この日には記録がありません」軽い表示)
- ペットフィルター切り替え:
  - 「すべて」選択中: すべての写真サムネイルを表示
  - 特定ペット選択中: そのペットが写った日のみサムネイル表示、他は空セル

**マーク仕様:**

- 🎀 記念日マーク: 記念日タグが付いた日(セル右上に小さく)
- ⭐ 掲載マーク: その日(またはその日の前日投稿)が「今日のペット」として掲載された日(セル左上に小さく)

**思い出ゾーン:**

カレンダーの上にボタンを並べる。v1.0 では「🎀 記念日を見る」のみ。将来「去年の今日」「Before / After」「月まとめ」を同じ位置に追加する。

## 4. ホーム(タブ)

**用途:** 今日の記録状況を見せる中心画面。

### 未記録時

```
┌────────────────────────────────────┐
│ (アプリ名)        [🐾アイコン] [⚙] │ ← フィルター + 設定
├────────────────────────────────────┤
│                                    │
│  [大きな写真]                       │ ← 思い出(過去の 1 枚)
│  2025年5月27日                      │
│  窓辺でひなたぼっこ                  │
│  メモ: ずっと寝てた                 │
│                                    │
├────────────────────────────────────┤
│                                    │
│   連続記録  12 日                    │
│                                    │
├────────────────────────────────────┤
│                                    │
│  [ 今日の 1 枚を残す ]               │ ← メイン CTA
│                                    │
└────────────────────────────────────┘
```

**思い出 1 枚の選び方(優先順):**

1. 1 年前(今日と同じ月日)の記録があればそれ
2. なければ過去の記録からランダムで 1 枚
3. 過去の記録がまったくないなら「これから始まる毎日を残しましょう」のような空状態メッセージ

**ペットフィルターによる絞り込み:**
- 「すべて」: 全記録から選ぶ
- 特定ペット: そのペットが写った記録から選ぶ
- 該当する記録がない場合は空状態メッセージ

**動作:**

- 思い出 1 枚タップ → 日別詳細 へ
- 「今日の 1 枚を残す」タップ:
  - アクションシートで「撮影」「アルバムから選択」を表示
  - 「撮影」 → カメラ起動 → 撮影後に写真記録フォームへ
  - 「アルバムから選択」 → 写真ライブラリピッカー → 選択後に写真記録フォームへ

### 記録済み時

```
┌────────────────────────────────────┐
│ (アプリ名)        [🐾アイコン] [⚙] │
├────────────────────────────────────┤
│                                    │
│  [大きな今日の写真]                  │
│  窓辺でひなたぼっこ                  │
│                                    │
│  メモ: 今日は1日中寝てた             │
│                                    │
│  🎀 おでかけ  🐱 ポチとタマ          │ ← 記念日タグ + 写っているペット
│                                    │
│  ⭐ 今日のペット参加中                │ ← 参加中の場合のみ表示
│                                    │
│  [編集]                             │
│                                    │
├────────────────────────────────────┤
│                                    │
│   連続記録  13 日                    │
│                                    │
└────────────────────────────────────┘
```

**動作:**

- 写真タップまたは「編集」タップ → 写真記録フォーム(編集モード) へ
- 翌日 0:00 を過ぎると「編集」ボタンは消える(閲覧のみ)

## 5. 今日のペット(タブ)

**用途:** その日に掲載されている 1 枚を表示。

### 候補ありの日

```
┌────────────────────────────────────┐
│ 今日のペット             [...] [🚩] │ ← 通報アイコン
├────────────────────────────────────┤
│                                    │
│  [大きな写真]                       │
│                                    │
│  窓辺でひなたぼっこ                  │
│  ポチとタマ                         │ ← タイトル / ペット名
│                                    │
├────────────────────────────────────┤
│                                    │
│  [かわいい 12] [きれい 4]            │ ← リアクションボタン
│  [かっこいい 3] [いいね 8]           │
│                                    │
└────────────────────────────────────┘
```

**表示する情報:**
- 写真
- タイトル
- ペット名(複数の場合は「と」で連結。例: 「ポチとタマ」「ポチとタマとミケ」)
- 各リアクションのカウント

**表示しない情報:**
- メモ
- 投稿者名
- ユーザーアイコン
- 投稿日時の詳細(掲載日のみ把握可能)

**リアクション:**

- 4 種類: かわいい / きれい / かっこいい / いいね
- 1 ユーザーが複数種類押せる
- 同じ種類は 1 回まで(2 回目押下は取り消し)
- 押した時点で楽観的更新(即座にカウント +1、ハイライト表示)
- 押下後にサーバーに非同期送信(オフラインでも操作可)
- 既に押したリアクションは強調表示

**通報ボタン:**
- 画面右上の旗アイコン
- 未ログイン時にタップ → ログイン画面に誘導
- ログイン時にタップ → 通報フォームへ

### 候補なしの日

```
┌────────────────────────────────────┐
│ 今日のペット                        │
├────────────────────────────────────┤
│                                    │
│  本日のペットは準備中です           │
│  明日の掲載候補を募集中です          │
│                                    │
└────────────────────────────────────┘
```

過去の掲載は閲覧できない(v1.0 では当日表示のみ)。

## 6. 写真記録フォーム

**用途:** 写真記録の作成・編集。

```
┌────────────────────────────────────┐
│ [キャンセル] 今日の記録 [保存]      │
├────────────────────────────────────┤
│                                    │
│  [大きな写真プレビュー]              │ ← タップで撮り直し
│                                    │
│  タイトル *                          │
│  [入力欄]                            │
│                                    │
│  メモ                                │
│  [複数行入力欄]                      │
│                                    │
│  写っているペット *                  │ ← Pro かつ複数ペット時のみ表示
│  [✓ ポチ] [□ タマ] [□ ミケ]        │
│                                    │
│  記念日タグ                         │
│  [+ タグを追加]                     │
│                                    │
│  今日のペットに参加                 │
│  [トグル: OFF]                      │
│  説明文(ON 時のみ表示):              │
│  この写真は確認後にアプリ内の「今日   │
│  のペット」として他のユーザーに表示  │
│  される可能性があります。            │
│  掲載日翌日にサーバーから削除されます│
│                                    │
└────────────────────────────────────┘
```

**入力項目:**

| 項目 | 必須/任意 | 制約 |
|---|---|---|
| 写真 | 必須 | 1 枚のみ |
| タイトル | 必須 | 最大 30 文字 |
| メモ | 任意 | 最大 200 文字 |
| 写っているペット | Pro かつ複数登録時に必須 | 最低 1 匹選択 |
| 記念日タグ | 任意 | 1 枚に 1 つだけ |
| 今日のペット参加 | 任意 | 初期 OFF |

**写っているペット選択:**

- 無料ユーザーまたは 1 ペットのみ登録の Pro ユーザー: この項目は UI に表示しない。自動的に唯一のペットを紐づける
- Pro で複数ペット登録時: チェックボックスで複数選択可能。デフォルトで「選択中のペット」が ON

**記念日タグ:**

- 「+ タグを追加」をタップでシート表示
- 候補: 誕生日 / うちの子記念日 / はじめて / おでかけ / その他(自由入力、最大 20 文字)
- 1 つ選んだら「+ タグを追加」ボタンが選択中タグ表示に変わる
- 選択中タグをタップで変更 or 削除

**今日のペット参加トグル:**

- 初期値 OFF
- 未ログインで ON にしようとすると → ログイン画面誘導
- ON にした瞬間に説明文が下に出る(削除タイミングについても明記)

**動作:**

- 「保存」タップ:
  - バリデーション(タイトル必須、ペット選択 Pro 時最低 1 匹)
  - ローカル保存(SQLite + FileSystem)
  - 連続日数更新ロジック(`docs/05-logic.md` 参照)
  - 「今日のペット参加」が ON なら送信キューに追加
  - フォームを閉じてホームへ
- 「キャンセル」タップ: 確認ダイアログ → フォーム破棄
- 編集モードの場合は当日のみ可能。翌日以降は閲覧画面として開く(後述「日別詳細」と同じ)

**編集モードでの追加 UI(参加中の場合のみ):**

```
今日のペット参加中
[ 参加を取り下げる ]
```

- pending の場合: 取り下げると Storage + DB から完全削除
- approved または scheduled の場合: 取り下げると status = withdrawn
- featured(掲載済)の場合: ボタン非表示(ユーザー取り下げ不可)

## 11. ペット追加・編集フォーム

ペット初期設定と同じ入力項目。追加と編集で同じフォームを使い回す。

編集時のみ「このペットを削除」ボタン(画面下部、赤文字)。

**削除時の挙動:**

- 確認ダイアログ「このペットを削除しますか?過去の記録のペット紐づけが解除されます」
- 削除実行 → ペット削除、過去の `entry_pets` 関連削除
- 写真記録自体は残る(ペット紐づけのみ消える)
- 最後の 1 ペットは削除不可(エラーダイアログ「最低 1 ペット必要です」)

## 15. 利用規約 / 16. プライバシーポリシー

**用途:** Web ビューまたは長文表示画面。

利用規約には以下を明記すること:

- 投稿写真は手動レビューを経て掲載される
- 「今日のペット」に参加した写真は、掲載日翌日にサーバーから削除される
- 掲載後の「掲載履歴」記録(写真コピー含む)は別領域に長期保存される
- 不適切な投稿への対応(通報・非表示・BAN)
- 個人情報保護方針

## 空状態とエラー状態

各画面で以下の空状態を必ず実装すること:

- カレンダー: 記録ゼロの月でもグリッドは正常表示、サムネイルが入らないだけ
- ホーム(未記録、思い出ゼロ): 「これから始まる毎日を残しましょう」
- 今日のペット(候補なし): 「本日のペットは準備中です」
- 記念日一覧(タグなし): 「特別な日に記念日タグを付けて、思い出を残しましょう」
- 掲載履歴(なし): 「まだ掲載された写真がありません」

詳細は `docs/09-edge-cases.md` を参照。
